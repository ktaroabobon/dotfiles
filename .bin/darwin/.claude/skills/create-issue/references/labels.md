# Available Labels

These are the existing labels in KURA-Family repositories. Select appropriate labels from this list. **Never create new labels.**

## Category Labels
- `api`: API関連
- `backend`: バックエンド関連
- `frontend`: フロントエンド関連
- `database`: データベース関連
- `infra`: インフラ関連
- `ci`: CI/CD関連
- `github-actions`: GitHub Actions関連
- `security`: セキュリティ関連

## Language / Tech Labels
- `golang`: Go言語関連
- `python`: Python関連
- `typescript`: TypeScript関連
- `typespec`: TypeSpec関連
- `revit-addin`: Revitアドイン関連

## Type Labels
- `bug`: Something isn't working
- `enhancement`: New feature or request
- `feature`: 新機能
- `documentation`: Improvements or additions to documentation
- `dependencies`: 依存関係の更新
- `migration`: Python→Go移行関連
- `mvp`: MVP達成に必要な機能
- `hotfix`: ホットフィックス
- `question`: Further information is requested

## Size Labels
- `size/XS`, `size/S`, `size/M`, `size/L`, `size/XL`

## Other Labels
- `tests`: テスト関連
- `good first issue`: Good for newcomers
- `help wanted`: Extra attention is needed

## Label Selection Guidelines

1. Always assign at least one **type label** (bug, enhancement, feature, etc.)
2. Assign relevant **category labels** (api, backend, database, etc.)
3. Assign a **language/tech label** if applicable (golang, python, etc.)
4. `migration` label applies to Python→Go migration work
5. Optionally assign a **size label** based on estimated effort
